Mapping Police Shootings

Summary

Web development skills don't just allow you to create slick web pages -- they enable you to communicate important information to broad audiences. In this exercise, you'll create an interactive map of the crowdsourced police shootings data from deadspin.

Objectives

By completing this challenge, you will practice and demonstrate mastery of the following skills:

  • Cloning a GitHub repository
  • Requesting data using an AJAX call
  • Creating an interactive map using Leaflet.js
  • Deploy a web site to a web server using git

Resources

We'll have a lecture that introduces the skills described above, and those should be used as a springboard to build a creative interactive visualization using the data provided.

Fork and Clone the Repo

You should now be comfortable forking and cloning a repository usig Gitub. This starting repository contains file stubs and the various assets you will need to complete the challenge.

  1. Go to my Police Shooting Repo on GitHub
  2. Click the "Fork" button located to the right of the repository name and follow any instructions you might receive
  3. In your forked repository on GitHub (not my original one), click the copy button next to the HTTPS clone URL to copy that URL to your clipboard
  4. Clone the repository to your local machine, just like you've done before.

Building your map

Your index.html file already has all of the necessary libraries and files included for you to begin building your map. However, it does not actually call the function that will build your map. Inside of the <script> tag of your index.html file, call the function that draws your map (drawMap).

Note, the drawMap function must be called after the <div id="container"> element is created in yourindex.html file. This is because the function depends on the existence of a container <div> element. Alternatively, in your drawMap function, you could first append a container <div> to build your map in, and reference that id.

In your map.js file, there are three empty functions already waiting for you. In the first function, drawMap, you should follow the code pattern demonstrated in class to build a map using Leaflet.js. Feel free to use the same maptiles we used in class (OpenStreetMap), or seek out some different styles from another map tile provider such as mapbox. Once you've completed this step, your maptiles should appear within the container div on your page.

In the second function, getData, you should write an AJAX request that gets data from the file data/response.json. This file contains an array of objects, one for each shooting incident. When the AJAX request is successful, you should call the third function, customBuild.

Note, the customBuild function must be called inside the data request, only once the AJAX call has completed successfully. This is because of the asynchronous nature of the request (i.e., your JavaScript code would keep running before the data was returned if you just wrote it in line).

Do something creative

Simply building the map and showing locations of police shootings with hover text is a good start to this challenge, though we cover all of those skills in class. Now is your chance to manipulate the map in novel ways to showcase a particular story in the data. There are lots of powerful ways to introduce and display the data in this dataset. First, think about the number of different dimensions of the data you have to work with:

  • Date of the incident ('Date Searched')
  • Summary of the incident ('Summary')
  • Link to a new story ('Source Link')
  • If the person shot was killed ('Hit or killed?')
  • The race of the person shot ('Race')
  • Many more, just look at the properties of the data...
While this is not a data visualization course, you can easily imagine the different ways in which you may want to show different variables of interest. Here are some ideas to get you started:

  • Changing the color or opacity of each marker based on race
  • Adding different layers to your map, based on race, outcome, or year.
  • Animating the display of each marker (i.e., bring them in one at a time, or by month -- this could be tricky...)
  • Changing the size of each marker based on the age of the person

Additional information

Showcasing sensitive data requires that you properly introduce the topic and site your sources. In addition to creating your map, your submitted webpage should include the following:

  • A link to the original data source
